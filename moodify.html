Moodify

Moodify is a Front End Web App that allows a user to submit their "mood", via facial recognition software or input, to return Spotify playlists which can be played in the webplayer. This project was built to display the use of AJAX calls, an understanding separate APIs, incorporation of Google Firebase, and building a responsive Web App with a CSS library.

How it works: We used the Kairos API to capture the user's mood with their facial recognition software. In order to accomplish this, our logic uses the device's camera to capture an image which is then sent to the Imgur API to return a weblink that the Kairos API can read. Once Kairos receives the image, we parse through the JSON to find the Emotion category which is given in another object that scores 6 different emotions. The emotion with the greatest value is the emotion we pass into the Spotify API to return 20 playlists. The playlists are displayed in a carousel and when double clicking on a playlist the Spotify Web Player opens in the page and the songs can be played (full song if signed into a Spotify account, otherwise a 30 second preview). If the camera is not utilized a search bar will send the input to the Spotify API to be used as well.

  • Kairos API: Facial Recognition
  • Spotify API: Retrieve Playlists and Songs
  • Imgur API: Send image links to Kairos
  • Google Firebase: Store Client Id and Client Secret in the logic
  • Materialize: CSS library for responsiveness

Additional Notes:

Since building and deploying this project, the Kairos API went from free usage to a paid subscription. We are looking into other facial recognition APIs to restore the camera usage.
